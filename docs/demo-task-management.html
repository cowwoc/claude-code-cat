Task Management

Structured AI development with hierarchical planning, automatic state tracking, and seamless workflow orchestration.

How It Works

CAT organizes your work into a clear hierarchy: Major versions contain minor versions, and minor versions contain individual tasks. This structure keeps even complex projects manageable and ensures nothing falls through the cracks.

Every task automatically tracks its own state, dependencies, and progress. When you complete a task, CAT knows which tasks are now unblocked and ready to work on next. No more manual tracking or wondering what to do next.

The /cat:work command handles everything: finding the next available task, setting up an isolated workspace, executing the plan, running quality reviews, and merging completed work. You focus on the code; CAT handles the orchestration.

Quick Example

terminal 
# Add a new task to the current version
$ /cat:add 2.0-implement-auth

# Check project status and see what's ready
$ /cat:status
v2.0 Progress: [=====>    ] 3/7 tasks
  - implement-auth (pending, deps: database-schema)
  - user-dashboard (in-progress)
  - api-endpoints (completed)

# Start working on the next available task
$ /cat:work
Loading task: 2.0-user-dashboard
Context loaded. Ready to continue implementation.
